# Consigne — Rapport d'analyse (rendu PDF)

Après analyse d'un ou plusieurs documents, produire un rapport suivant **cette structure exacte**. Sortie = HTML rempli sur le gabarit `_templates/rapport-style.html`, que l'utilisateur imprime en PDF (Ctrl+P → « Enregistrer en PDF »).

## Règles de fond (non négociables)
- **Citer systématiquement fichier + article** pour chaque affirmation : `[CCN 3029, art. 12]`, `[C. trav., L.3121-64]`.
- **Jamais de qualification juridique d'un cas individuel** → renvoi juriste fédération.
- Ce qui n'est pas dans le corpus : le dire. Ne rien déduire ni inventer.
- Hiérarchie des normes + **règle de faveur** (disposition la plus favorable au salarié prime).
- Signaler le statut (projet / en vigueur / abrogé) de chaque document analysé.

## Structure obligatoire du rapport

1. **En-tête** — titre, document(s) analysé(s), date, mention « Document de travail syndical — ne constitue pas un conseil juridique ».
2. **Synthèse** — 5 lignes max : ce qui est conforme, ce qui pose problème, l'urgence.
3. **Tableau des risques** — une ligne par point analysé :
   | Point | Conforme | Risque faible | Risque moyen | Risque élevé |
4. **Analyse détaillée** — pour chaque point :
   - Ce que dit le document
   - Ce qu'exige le Code du travail `[réf.]`
   - Ce qu'exige la CCN `[réf.]`
   - Écart / risque (nullité, prud'hommes)
5. **Hiérarchie des normes** — quelle règle s'applique par sujet, en appliquant la règle de faveur.
6. **Recommandations** — actions concrètes pour la négociation / défense, priorisées.
7. **Sources citées** — liste des fichiers + articles référencés.
8. **Pied** — « Établi le AAAA-MM-JJ · Corpus : … · Vérifier toute référence sur Légifrance avant usage contentieux. »

## Code couleur risque (dans le gabarit)
- Vert = conforme · Orange = risque moyen · Rouge = risque élevé.
